**Key Features:**
- 🌐 Runs diagnostics over standard Ethernet/IP networks
- 🔄 Enables remote and fast vehicle diagnostics
- 🛠️ Supports UDS (Unified Diagnostic Services) over TCP/IP
- 🚦 Allows simultaneous diagnostics for multiple ECUs
- 🔒 Security support (authentication, encryption possible)

**Physical/Electrical Layer:**
- Uses standard automotive Ethernet (100BASE-T1, 1000BASE-T1)
- Operates over IP (IPv4/IPv6)

**Protocol Stack:**
- Sits above TCP/IP stack
- Often used with UDS (ISO 14229) and ISO 13400

**Topology:**
- Star or switched Ethernet network in modern vehicles

**Use Cases:**
- Remote diagnostics by OEMs and service centers
- Over-the-air (OTA) updates and troubleshooting
- Workshop diagnostics (faster than CAN-based)

**Pros:**
- 🚀 High speed (compared to CAN diagnostics)
- 🌍 Remote access possible
- 🧩 Scalable to many ECUs

**Cons:**
- 🔐 Security is critical (risk of remote attacks)
- 🏗️ Requires modern Ethernet infrastructure

**Standard:**
- ISO 13400 (DoIP)

**Typical Applications:**
- Next-gen vehicles with Ethernet backbones
- Electric vehicles, ADAS, connected cars

**Memorization Tip:**
<span style="color:#1976d2;">Think of "DoIP" as "Diagnose Over Internet Protocol"—just like you can check your car from anywhere, thanks to the internet!</span>
