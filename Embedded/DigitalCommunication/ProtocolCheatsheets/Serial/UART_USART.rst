UART / USART
============

**Domain:** Embedded/Industrial

**Description:**
Simple serial, point-to-point communication.

.. raw:: html

	<div style="background-color:#e3f2fd; border-radius:10px; padding:16px; margin-bottom:16px;">

**Key Features:**
- 💬 Simple, asynchronous or synchronous
- 🔄 Point-to-point, full/half duplex
- 🛠️ Configurable baud rate, parity, stop bits
- 🧩 Widely supported in microcontrollers

**Physical/Electrical Layer:**
- 🔌 TTL/CMOS logic, RS-232/485 compatible
- ⚡ Data rates: up to several Mbps

**Topology:**
- 🌐 Point-to-point

**Use Cases:**
- 🖥️ Debugging, console, GPS, modems
- 🛠️ Board-to-board, chip-to-chip

**Pros:**
- 🧩 Simple, low cost, easy to use
- 💬 Flexible, widely supported

**Cons:**
- 🏗️ No addressing, only two devices
- ⚠️ No error correction

**Standard:**
- 📜 EIA/TIA-232/485, vendor-specific

**Typical Applications:**
- 🖥️ Embedded systems, serial consoles

**Memorization Tip:**
<span style="color:#1976d2;">Think of UART as the "universal chat line" for chips—simple, direct, and always ready to talk!</span>
