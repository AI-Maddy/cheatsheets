=====
DDS
=====

.. note::
   This is a cheatsheet for DDS protocol.

**DDS**
- Data Distribution Service
- Real-time, publish/subscribe middleware
- Used in robotics, defense, IIoT

**Key Features**
- Decentralized, scalable
- QoS policies, discovery
- Reliable multicast, low latency

**Mnemonic**: "Distributed Data, Superfast!"

**Common Use Cases**
- Autonomous vehicles, industrial control, simulation

**Pros**
- Real-time, scalable, robust
- Flexible QoS

**Cons**
- Complex configuration
- Larger footprint than MQTT/CoAP
